What is a Function?

A function is a block of organized, reusable code that performs a single, related action. In R, there are two types of functions:

Example 1: A Simple Addition Function

add_numbers <- function(x, y) {
  result <- x + y
  return(result)
}

The function add_numbers() takes two inputs and returns their sum.

Default Arguments

You can also assign default values to arguments. If a value is not provided by the caller, it uses the default. 

power <- function(base, exponent = 2) {
  return(base ^ exponent)
}

power(3)        # 3^2 = 9
power(3, 3)     # 3^3 = 27

Variable Number of Arguments (…)

The ellipsis ... lets you pass any number of arguments to a function. 

sum_all <- function(...) {
  numbers <- c(...)
  return(sum(numbers))
}
sum_all(1, 2, 3, 4)  # Output: 10

This feature is particularly useful when you are unsure how many arguments might be passed.

Scope of Variables

Local vs Global Scope: Variables defined inside a function are local â€“ they do not affect the global environment. 

my_function <- function() {
  x <- 10  # local variable
  return(x)
}

x <- 5  # global variable
my_function()  # Output: 10
x              # Still 5

To change a global variable inside a function, you would need the << - operator (though it's not recommended for clean, organized code).

Returning Multiple Values

R allows returning lists or vectors to simulate returning multiple values from a function. 

math_ops <- function(x, y) {
  add <- x + y
  mult <- x * y
  return(list(sum = add, product = mult))
}

result <- math_ops(4, 5)
result$sum      # 9
result$product  # 20

Example: A Mini Calculator Function

calculator <- function(a, b, operation = "add") {
  if (operation == "add") {
    return(a + b)
  } else if (operation == "subtract") {
    return(a - b)
  } else if (operation == "multiply") {
    return(a * b)
  } else if (operation == "divide") {
    if (b != 0) {
      return(a / b)
    } else {
      return("Cannot divide by zero")
    }
  } else {
    return("Invalid operation")
  }
}
calculator(10, 2, "multiply")  # Output: 20

Why Use Functions?

  • Reuse code: Write once and use multiple times.
  • Keep code clean: Functions help organize your code into modular sections.
  • Debug and test easily: Isolated functions simplify debugging.
  • Modular programming: Break down complex problems into smaller, more manageable parts.

Pro Tip: Anonymous Functions

Anonymous functions are unnamed and useful for one-time operations, especially when used with functions like those in the apply() family. 

(function(x) x^2)(5)  # Output: 25
