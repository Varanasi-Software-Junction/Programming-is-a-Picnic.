Key Concepts

1. Models

Models define the structure of your database. In the blogging site, we used:

  • CustomUser: Extends Django's AbstractUser to manage roles (admin, writer, reader).
  • BlogPost: Represents a blog post, with fields for the title, content, author, etc.

Verification: After defining models, run:

python manage.py makemigrations
python manage.py migrate

Check the Django admin to see the new models listed.

2. Forms

Django forms handle user input. We used ModelForm to easily create forms from models.

Verification: Ensure the form renders correctly on the create_blog.html page.

3. Views

Views handle the logic for rendering data to templates. We used class-based views (CBVs) like CreateView and ListView.

Verification: Ensure users are redirected based on their roles (admin, writer, reader) and check page access accordingly.

4. Templates

Templates in Django are used to render HTML. We have two templates: blog_list.html and create_blog.html.

Verification: Load the templates in a browser to ensure they display correctly and include all the necessary information.

5. User Authentication and Permissions

Django’s built-in authentication system manages login, logout, and role-based permissions.

Verification: Test login/logout functionality and access controls for different user roles (admin, writer, reader).

6. MySQL Database Setup

The project uses MySQL as its database. We installed the mysqlclient package to interact with MySQL in Django.

Verification: Use the MySQL shell to confirm that tables like blog_blogpost and auth_user exist.

Detailed Explanation of Steps

Follow these steps to ensure your project runs smoothly:

  1. Project Setup: Create a new Django project using django-admin startproject travel_blog.
  2. App Creation: Use python manage.py startapp blog to create the blog app.
  3. Database Configuration: Ensure the MySQL database is connected properly in settings.py.
  4. Custom User Model: Create the CustomUser model and verify roles (admin, writer, reader).
  5. Views and Permissions: Ensure that the correct users can view and create blog posts.
  6. Form Rendering: Check that the BlogPostForm displays correctly in the template.
  7. Template Rendering: Load the blog_list.html and create_blog.html templates in a browser.

Things to Verify After Each Step

  • Project Setup: Run the server and ensure the Django project loads in the browser.
  • Database Configuration: Check for successful migrations.
  • Custom User Model: Verify user roles in the Django admin panel.
  • Templates: Test rendering of templates and functionality for each user role.
  • Views: Ensure role-based access control works as expected.
