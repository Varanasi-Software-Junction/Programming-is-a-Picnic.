Important Flutter & Dart Commands

Flutter Commands

1. flutter create

Creates a new Flutter project.

flutter create <project_name> --org com.example --project-name custom_name --template app --platforms web

Options:

  • --dry-run: Simulates the command without actually creating the project.
  • --force: Forces overwriting if the directory already exists.

2. flutter run

Runs the app on a connected device.

flutter run -d chrome --release

Options:

  • --flavor: Specifies which flavor to run, e.g., `staging` or `production` builds.
  • --verbose: Outputs more detailed logs.
  • --trace-startup: Enables tracing of startup performance.

3. flutter pub get

Fetches all dependencies listed in the pubspec.yaml file.

flutter pub get

4. flutter upgrade

Upgrades Flutter to the latest stable version.

flutter upgrade --force

Options:

  • --force: Forces the upgrade, even if no new versions are available.

Dart Commands

1. dart run

Runs a Dart program.

dart run <script.dart>

Options:

  • --observe: Runs the Dart script in debug mode with observatory enabled.

2. dart create

Creates a new Dart project.

dart create <project_name>

Options:

  • --force: Overwrites any existing directory.

3. dart pub get

Fetches dependencies for the Dart project.

dart pub get

4. dart analyze

Analyzes the Dart code for potential issues.

dart analyze

5. dart test

Runs tests for the Dart project.

dart test

Options:

  • --name <pattern>: Runs only tests whose names match the provided pattern.
  • --concurrency: Limits the number of concurrent test suites.

6. dart format

Formats Dart code according to the style guidelines.

dart format . --fix

Options:

  • --dry-run: Shows which files would be formatted, without applying the changes.
  • --fix: Applies all possible fixes as part of formatting.

7. dart compile

Compiles a Dart script to an executable.

dart compile exe <script.dart> --output=my_executable

Options:

  • --output: Specifies the output file name for the compiled executable.

8. dart doc

Generates documentation for a Dart project.

dart doc .

Command Sequences

1. Create and Run a Basic Flutter Project (Android, iOS, Web, Desktop)

flutter create my_project --platforms web
cd my_project
flutter pub get
flutter run
flutter build apk --release
flutter build ios --release
flutter run -d ios
flutter run -d chrome
flutter build web
flutter run -d windows

2. Clean, Build, and Run an Optimized Production Build (Android)

flutter clean
flutter pub get
flutter build apk --release --verbose
flutter install

3. Debugging a Flutter App (Attach to Running Instance)

flutter run --debug
flutter logs
flutter attach -d <device_id>
dart analyze

4. Managing Dependencies and Running Tests

flutter pub add provider
flutter pub get
flutter pub outdated
flutter test --coverage

5. Build a Flutter App for Multiple Platforms (Android, iOS, Web, Desktop)

flutter build apk --release
flutter build ios --release
flutter build web
flutter build windows

6. Project Setup with Version Control (Git)

flutter create my_project
cd my_project
git init
git add .
git commit -m "Initial commit"
flutter run

7. Prepare and Build an App Bundle (AAB) for Google Play Store

flutter clean
flutter pub get
flutter build appbundle --release
dart analyze
flutter run --release

8. Customizing and Running in Different Modes (Debug, Release, Profile)

flutter run --debug
flutter run --profile
flutter run --release
flutter build apk --release
flutter attach

9. Managing the Flutter Environment (Upgrades, Configs)

flutter doctor
flutter upgrade --force
flutter config --enable-web
flutter precache
flutter pub cache repair
