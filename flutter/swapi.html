Flutter SWAPI Films App Walkthrough

This guide details the steps to build a Flutter app that fetches, stores, and displays SWAPI film data with offline capabilities using SQLite.

Step 1: Create the Project Structure

Tasks:

  • Initialize Flutter project and add dependencies (`connectivity_plus`, `http`, `sqflite`, `path`).

Validation:

  • Ensure `flutter pub get` runs without errors.
  • Confirm project folders for `models`, `services`, and `database` are created.

Step 2: Define the Film Model

Tasks:

  • Create `film.dart` in `models` folder and define model with properties.
  • Add methods to convert to/from JSON and SQLite map format.

Validation:

  • Check if `Film` instances can convert to/from JSON and SQLite map format.

Step 3: Set Up SQLite Database (DatabaseHelper)

Tasks:

  • Initialize database and create `films` table in `database_helper.dart`.
  • Implement CRUD operations.

Validation:

  • Confirm that SQLite operations work with test data.

Step 4: Set Up API Service (ApiService)

Tasks:

  • Define base URL and implement fetch and save functionality.

Validation:

  • Verify API response and confirm data is saved or updated in SQLite.

Step 5: Set Up Internet Connectivity Check

Tasks:

  • Implement internet status check and update SQLite when connected.

Validation:

  • Confirm that the app detects connectivity status correctly.

Step 6: Build the Home Screen (home_screen.dart)

Tasks:

  • Create `HomeScreen` widget and load data from SQLite.

Validation:

  • Ensure films display correctly and check error messages for offline mode.

Step 7: Test the Complete Application

Tasks:

  • Test app functionality with and without internet connectivity.

Validation:

  • Confirm all functionalities work as expected both online and offline.
