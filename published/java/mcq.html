Online MCQ Test Project Requirements

Project Overview

An online platform for an institute to create and conduct MCQ-based tests for students. It will have user roles for admin and students. The platform should include functionalities to manage tests, display results, and allow students to take tests in a simple, user-friendly interface.

Core Technologies

  • Frontend: React
  • Backend: Node.js with Express
  • Database: MySQL

Functional Requirements

User Roles

  • Admin: Manage MCQ tests, view and download results, manage student accounts.
  • Student: Login, view available tests, take tests, view individual test results.

Features

Admin Features

  • Manage tests (Create, Edit, Delete).
  • Manage students (Add, Edit, Delete).
  • View and export results to CSV.

Student Features

  • View list of available tests.
  • Take tests with timer and submit answers.
  • View results after submitting the test.

Technical Requirements

Frontend (React)

  • Use React Router for navigation between pages.
  • Components: Login Page, Admin Dashboard, Test Management Page, Test Taking Interface, Results Page.
  • Use Material UI or Bootstrap for styling.

Backend (Node.js with Express)

  • REST API with endpoints for user authentication, test management, submitting answers, and fetching results.

Database (MySQL)

  • Tables: users, tests, questions, results.

Pages & Navigation

  • Admin Pages: Login Page, Dashboard, Test Management Page, Question Management Page, Results Page.
  • Student Pages: Login Page, Dashboard (List of Tests), Test Page, Results Page.

Additional Features (Optional)

  • Password Reset functionality.
  • Pagination for test and result lists.
  • Responsive Design for mobile and desktop compatibility.

Deployment Requirements

Project Timeline

  • Week 1: Set up project structure, design database schema, build basic UI and routes in React.
  • Week 2: Develop backend APIs for user authentication and test management, integrate frontend with backend APIs.
  • Week 3: Implement test-taking functionality with a timer, result calculation, and display.
  • Week 4: Test the application, fix bugs, and deploy the application.

Deliverables

  • Complete source code.
  • Deployed application link.
  • Basic documentation on how to set up and run the project locally.
